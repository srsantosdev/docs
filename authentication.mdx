<Info>
  **Pré-requisito**: Gere sua chave de API no painel da plataforma do Scanify. Ela será usada para autenticar todas as requisições.
</Info>

## Como autenticar

Todas as requisições à API do Scanify exigem autenticação via header `Authorization`, no formato **Bearer**.

```bash
-H "Authorization: Bearer SUA_CHAVE_DE_API"
```

<Warning>
  **Nota**: A chave de API é única e permanente para sua conta (a menos que você regenere).
</Warning>

## Exemplo com cURL


Inclua o token JWT no header `Authorization` de cada requisição aos endpoints protegidos da API:

```bash
curl -X POST https://api.scanify.com.br/extract \
  -H "Authorization: Bearer sk_live_abc123..." \
  -F "file=@nfe.pdf" \
  -F "callback_url=https://seudominio.com/webhook"
```

## Boas práticas

- Nunca exponha sua chave de API em ambientes públicos ou código client-side.
- Prefira armazená-la em variáveis de ambiente.
- Se necessário, você pode regenerar a chave diretamente na plataforma.

## Resolução de Problemas

Confira abaixo algumas soluções para problemas comuns relacionados à autenticação na API do Scanify:

<AccordionGroup> 
  <Accordion title="Erro 401 - Unauthorized"> Esse erro indica que a chave de API está ausente ou incorreta. Verifique se o header `Authorization` está no formato correto: `Bearer SUA_CHAVE_DE_API`. </Accordion> 
  <Accordion title="403 - Conta inativa"> Sua conta pode estar desativada ou com uso suspenso. Verifique seu status no painel da plataforma. </Accordion> 
  <Accordion title="Erro 429 - Limite excedido"> Você atingiu o número máximo de documentos no seu plano atual. Faça upgrade ou aguarde a renovação do ciclo mensal. </Accordion> 
</AccordionGroup>

<Info>

</Info>
